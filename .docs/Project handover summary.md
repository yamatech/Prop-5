# 📋 VSTプラグイン開発プロジェクト「prop-5」引継ぎ用サマリー

## 1. プロジェクト概要

- **プロジェクト名:** prop-5
- **目標:** Sequential Circuits「Prophet-5」をモデルとした、JUCEフレームワークに基づくポリフォニックVST3/AUシンセサイザープラグインの開発。
- **開発方針（最重要）:** > **「回路の挙動やDSP（音の処理）は100%実機（本物）を目指し、実装しやすいところから効率的に進める」**
このハイブリッド方針に基づき、これまで4日間の集中開発を経て、音源としての心臓部（DSP・シグナルフロー）はバグ修正も含めて完全に構築・駆動しています。

## 2. 現在の実装ステータス（DSP / シグナルフロー完了済み）

音響処理ロジックは実機の仕様を精密にモデリングし、安定動作しています。

- **ボイス構成:** 5ボイス・ポリフォニック仕様。
- **オシレーター部:** OSC A および OSC B を搭載。各種波形選択、ピッチ（Frequency）コントロールを実装。
- **フィルター部（VCF）:** アナログの質感を意識した4ポール・ラダーフィルター（ローパス）をモジュレーション対応で実装。
- **エンベロープ（ENV）:** VCA用（音量）とVCF用（フィルター）の2系統のADSRエンベロープを完全実装。
- **LFO（自作モジュレーション源）:** Prophet-5固有の仕様に準拠したLFO回路。
- **Poly Mod（ポリ・モッド）システム:** 実機の最大の特徴である、OSC BとVCF ENVをソースとして、OSC A（Freq/PW）やVCF（Cutoff）に変調をかけるマトリクス機構を完全再現。
- **OSC B Lo Mode:** OSC Bを低周波発振器（LFO第二系統）として切り替えて動作させるモードを実装。Poly Modとの連携による複雑な金属音・効果音の生成に対応。

## 3. データ・パラメーター管理（APVTS構造）

- すべてのシンセ・パラメーターは、JUCEの `juce::AudioProcessorValueTreeState`（以下APVTS）を使用して一元管理されています。
- 開発の過程で、LFOやPoly Modのパラメータ追加に伴うリファクタリングを実施済みです。パラメーターIDや内部グループ化は綺麗に整理され、バグやビルドエラーはすべて解消されています。

## 4. 次フェーズの依頼タスク（GUI & プリセット実装）

音源側は完成しているため、ここからは「見た目の構築」**と**「楽器としての実用性向上」の2点に集中してください。

### タスク①：本格的なカスタムGUIの構築

現在、画面はDAWが自動生成する簡易的な汎用UI（スライダーの羅列）です。これをProphet-5の実機を彷彿とさせるデザインに仕上げてください。

- **レイアウト:** `PluginEditor.h / .cpp` を編集し、実機のコントロールパネル（OSC、FILTER、POLY MOD、LFOなど）の配置を踏襲したレイアウトを設計する。
- **カスタムデザイン（LookAndFeel）:** ヴィンテージ風のロータリーノブ、クラシックな四角いボタンスイッチ、ダークブラウンのメインパネル、木目調のウッドフレームなどをJUCEの2Dグラフィックス描画、または画像アセット（使用する場合）を用いて美しく再現する。
- **パラメーター同期:** 各UIコンポーネント（Slider, Button等）を、APVTSの対応するパラメーターへ `juce::AudioProcessorValueTreeState::SliderAttachment` や `ButtonAttachment` を用いてスレッドセーフに接続・同期する。

### タスク②：プリセット管理システム（状態保存）の実装

DAW上で音色を保存・復元し、プラグインとして完結させます。

- `PluginProcessor.cpp` 内の `getStateInformation()` および `setStateInformation()` を実装。APVTSの現在のValueTree状態をXML（またはバイナリ）形式でシリアライズ/デシリアライズし、ホストDAWのプロジェクト保存時に音色状態が100%復元されるようにする。
- プラグイン起動時のデフォルト音色（Init Patch）の設定、および簡単なプリセットバンク（数種類を切り替えられる仕組み）の搭載。

## 5. 新エージェントへの注意事項

- **DSPロジックの保護:** 音色のシグナルフローやPoly Mod等の変調ロジックはデバッグを含めて完了しています。GUIやプリセット実装に伴う変更を除き、`PluginProcessor` 内の音声信号処理ループ（`processBlock`）のコアロジックを不用意に変形させないでください。
- **コードの同時レビュー:** 各コンポーネントを実装する際は、全体のソースコードを随時見渡し、現在のAPVTSのパラメーター名（ID）と完全に一致しているか、メモリリークがないかを相互に確認しながら、効率的かつ確実に進めてください。
